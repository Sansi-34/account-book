# 账号本

一个本地优先的轻量账号密码管理工具。它适合保存常用网站、账号、密码、标签和备注，不需要登录账号，也不会把数据上传到服务器。

> 当前项目没有经过专业安全审计。适合个人本地使用，不建议替代成熟的专业密码管理器。

## 功能

- 本地保存网址、账号、密码、标签和备注
- 用打开密码解锁数据
- 粘贴整段登录信息后一键识别填入
- 搜索记录
- 复制账号、密码、网址
- 一键打开网址并复制密码
- 双击列表记录直接打开网址
- 复制后 30 秒尝试清空剪贴板
- 生成随机密码
- 导出和恢复加密备份
- 10 分钟无操作自动锁定
- 直接打开 `index.html` 即可使用

## 快速使用

直接双击打开：

```text
index.html
```

第一次打开时，输入一个打开密码。以后需要用同一个密码解锁。

如果你更喜欢用本地服务预览，也可以在项目目录运行：

```bash
python -m http.server 8000
```

这一步不是必须的。

## 数据保存在哪里

账号本的数据保存在当前浏览器的 `localStorage` 里，键名是：

```text
local-account-vault-v1
```

保存前会先在浏览器里加密。清理浏览器数据、换浏览器、换电脑，都可能导致本地数据不可见，所以建议定期点击“备份”导出加密 JSON 文件。

## 加密说明

项目使用浏览器 Web Crypto API：

- PBKDF2-SHA256
- 250000 次迭代
- AES-GCM-256
- 每个账号本使用随机 salt
- 每次保存使用新的随机 IV

打开密码不会被保存。忘记打开密码后，已加密的数据无法恢复。

## 安全体验

- 页面解锁后，10 分钟无操作会自动锁定。
- 复制账号、密码或网址后，会在 30 秒后尝试清空剪贴板。
- 如果这 30 秒内你复制了别的内容，账号本不会覆盖新的剪贴板内容。
- 导入备份时会先检查必要字段，再尝试解密。

## 快捷操作

- `Ctrl+S`：保存当前记录
- `Ctrl+N`：新建记录
- `Ctrl+K`：聚焦搜索框
- `Ctrl+Shift+L`：锁定
- 双击左侧记录：打开该记录的网址

## 一键识别

可以把一整段登录信息粘到“一键识别”输入框，再点“识别填入”。识别在本地完成，不会联网。

支持常见字段：

```text
名称：GitHub
网址：https://github.com
账号：sansi@example.com
密码：S3cret!123
备注：主账号
```

如果没有明确标签，也会尝试识别 URL、邮箱和手机号。

## 安全边界

这个项目会尽量保持简单、透明，但它不是万能的：

- 不提供云同步
- 不提供找回密码
- 不防止电脑已经中毒、浏览器被恶意扩展读取、系统被远程控制等情况
- 不建议在公共电脑或不可信设备上使用
- 没有经过第三方安全审计
